--- title: Interpolating EBSD Data last_updated: 11-Sep-2020 sidebar: documentation_sidebar permalink: EBSDInter.html folder: documentation toc: false --- Interpolating EBSD Data edit page

In the section Denoising and Filling Missing Data we have discussed how to work with noisy EBSD data the contained non indexed pixels. Hereby, we made the assumption that the grid before and after the operations is the same.

In this section we explain how to interpolate an EBSD map at positions that do not belong to the grid. Lets us consider a simple example

{% highlight matlab %} mtexdata twins; [grains, ebsd.grainId] = calcGrains(ebsd('indexed')); plot(ebsd('indexed'),ebsd('indexed').orientations) {% endhighlight %}
{% include inline_image.html file="EBSDInter_01.png" %}

As this functionality is currently only available for EBSD data on a rectangular grid we have to apply the command gridify first

{% highlight matlab %} ebsd = ebsd.gridify; ebsd = ebsd.project2FundamentalRegion(grains); {% endhighlight %}

Now we can use the command interp to interpolate the orientation at arbitrary coordinates x and y.

{% highlight matlab %} x = 30.5; y = 5.5; e1 = interp(ebsd,x,y) {% endhighlight %} {% highlight plaintext %} e1 = EBSD Phase Orientations Mineral Color Symmetry Crystal reference frame 1 1 (100%) Magnesium LightSkyBlue 6/mmm X||a*, Y||b, Z||c Id Phase phi1 Phi phi2 x y bands bc bs error mad grainId oldId 1 1 163 112 186 30.5 5.5 10 160 255 0 0.4 36 3109 Scan unit : um {% endhighlight %}

By default the command interp performs inverse distance interpolation. This is different to

{% highlight matlab %} e2 = ebsd('xy',x,y) {% endhighlight %} {% highlight plaintext %} e2 = EBSD Phase Orientations Mineral Color Symmetry Crystal reference frame 1 1 (100%) Magnesium LightSkyBlue 6/mmm X||a*, Y||b, Z||c Id Phase phi1 Phi phi2 bands bc bs error mad x y grainId oldId 13993 1 163 112 186 10 160 255 0 0.4 30.6 5.4 36 3109 Scan unit : um {% endhighlight %}

which returns the nearest neighbour EBSD measurement. Lets have a look at the difference

{% highlight matlab %} angle(e1.orientations,e2.orientations)./degree {% endhighlight %} {% highlight plaintext %} ans = 0.1508 {% endhighlight %}

Change of the measurement grid

The command interp can be used to evaluate the EBSD map on a different grid, which might have higher or lower resolution or might even be rotated. Lets demonstrate this

{% highlight matlab %} % define a rotated coarse grid omega = 5*degree; x = linspace(ebsd.xmin-cos(omega)*ebsd.ymax,ebsd.xmax,100); y = linspace(ebsd.ymin-sin(omega)*ebsd.xmax,ebsd.ymax,50); [x,y] = meshgrid(x,y); xy = [cos(omega) -sin(omega); sin(omega) cos(omega) ] * [x(:),y(:)].'; % define the EBSD data set on this new grid ebsdNewGrid = interp(ebsd,xy(1,:),xy(2,:)) % plot the regridded EBSD data set plot(ebsdNewGrid('indexed'),ebsdNewGrid('indexed').orientations) {% endhighlight %} {% highlight plaintext %} ebsdNewGrid = EBSD Phase Orientations Mineral Color Symmetry Crystal reference frame 0 2624 (52%) notIndexed 1 2376 (48%) Magnesium LightSkyBlue 6/mmm X||a*, Y||b, Z||c Properties: x, y, bands, bc, bs, error, mad, grainId, oldId Scan unit : um {% endhighlight %}
{% include inline_image.html file="EBSDInter_02.png" %}

Note, that we have not rotated the EBSD data but only the grid. All orientations as well as the position of all grains remains unchanged.
