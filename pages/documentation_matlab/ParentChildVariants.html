--- title: Parent Child Variants last_updated: 21-Mar-2021 sidebar: documentation_sidebar permalink: ParentChildVariants.html folder: documentation toc: false --- Parent Child Variants edit page

The crystallographic structure of most materials is dependend on external conditions as temperature and pressure. When the external conditions change the crystals may undergo a phase transition from the inital phase, often called parent phase, into the child phase. While both phases still have the same chemical composition their crystallographic structure might be quite different. A typical example are the alpha and beta phase of titanium. While the parent beta phase is cubic

{% highlight matlab %} csBeta = crystalSymmetry('432',[3.3 3.3 3.3],'mineral','Ti (beta)'); {% endhighlight %}

the child alpha phase is hexagonal

{% highlight matlab %} csAlpha = crystalSymmetry('622',[3 3 4.7],'mineral','Ti (alpha)'); {% endhighlight %}

Let oriParent

{% highlight matlab %} oriParent = orientation.rand(csBeta); {% endhighlight %}

be the orientation of the atomic lattice befor phase transition and oriChild the orientation of the atomic lattice after the phase transition. Since during a phase transition the atoms reorder with respect to a minimal energy constraint, both orientations oriParent and oriChild are in a specific orientation relationship with respect to each other. In the case of alpha and beta Titanium the dominant orientation relationship is described by the Burger orientation relationship

{% highlight matlab %} beta2alpha = orientation.Burger(csBeta,csAlpha) {% endhighlight %} {% highlight plaintext %} beta2alpha = misorientation crystal symmetry : Ti (beta) (432) crystal symmetry : Ti (alpha) (622, X||a*, Y||b, Z||c*) Bunge Euler angles in degree phi1 Phi phi2 Inv. 185.264 90 45 0 {% endhighlight %}

A corresponding child orientation would then be

{% highlight matlab %} oriChild = oriParent * inv(beta2alpha) {% endhighlight %} {% highlight plaintext %} oriChild = orientation crystal symmetry : Ti (alpha) (622, X||a*, Y||b, Z||c*) Bunge Euler angles in degree phi1 Phi phi2 Inv. 227.991 81.6844 337.409 0 {% endhighlight %}

This orientation relationship is characterised by alignment of hexagonal (0001) plane with the cubic (110) plane and alignment of the hexagonal [2-1-10] direction with the cubic [-11-1] direction.

{% highlight matlab %} hAlpha = Miller({0,0,0,1},{2,-1,-1,0},csAlpha); hBeta = Miller({1,1,0},{1,1,1},csBeta); plotPDF(oriParent,hBeta,'MarkerSize',20,'MarkerFaceColor','none','linewidth',4) nextAxis(1) hold on plot(oriChild * hAlpha(1).symmetrise,'MarkerSize',12) xlabel(char(hAlpha(1)),'color',ind2color(2)) nextAxis(2) plot(oriChild * hAlpha(2).symmetrise,'MarkerSize',12) xlabel(char(hAlpha(2)),'color',ind2color(2)) hold off drawNow(gcm) {% endhighlight %}
{% include inline_image.html file="ParentChildVariants_01.png" %}

We could also use these alignment rules to define the orientation relationship as

{% highlight matlab %} beta2alpha = orientation.map(Miller(1,1,0,csBeta),Miller(0,0,0,1,csAlpha),... Miller(-1,1,-1,csBeta),Miller(2,-1,-1,0,csAlpha)); {% endhighlight %}

The advantage of the above definition by the alignment of different crystal directions is that it is independent of the convention used for the hexagonal crystal coordinate system.

Child Variants

Due to crystal symmetry each orientation of a parent beta grain has 24 different may transform into up to 24 child orientations.

{% highlight matlab %} oriParentSym = oriParent.symmetrise {% endhighlight %} {% highlight plaintext %} oriParentSym = orientation size: 24 x 1 crystal symmetry : Ti (beta) (432) {% endhighlight %}

Applying the beta2alpha phase relationship to these 24 different represenations we obtain 24 child orientations.

{% highlight matlab %} oriChild = oriParentSym * inv(beta2alpha) {% endhighlight %} {% highlight plaintext %} oriChild = orientation size: 24 x 1 crystal symmetry : Ti (alpha) (622, X||a*, Y||b, Z||c*) {% endhighlight %}

Some of these child orientations are symmetrically equivalent with respect to the hexagonal child symmetry. In fact there are 12 pairs of symmetrically equivalent child orientations as depicted in the following inverse pole figure.

{% highlight matlab %} plotIPDF(oriChild,vector3d.Z) {% endhighlight %}
{% include inline_image.html file="ParentChildVariants_02.png" %}

These 12 pairs are called the variants of the parent orientation oriParent with respect to the orientation relation ship beta2alpha. They can be computed more directly using the command variants.

{% highlight matlab %} oriChild = variants(beta2alpha,oriParent); for i = 1:12 plotIPDF(oriChild(i),ind2color(i),vector3d.Z,'label',i,'MarkerEdgeColor','k'); hold on end hold off {% endhighlight %}
{% include inline_image.html file="ParentChildVariants_03.png" %}

As we can see each variant can be associated by a variantId. You can pick specific variants by their variantId using the syntax

{% highlight matlab %} oriChild = variants(beta2alpha,oriParent,2:3) {% endhighlight %} {% highlight plaintext %} oriChild = orientation size: 1 x 2 crystal symmetry : Ti (alpha) (622, X||a*, Y||b, Z||c*) Bunge Euler angles in degree phi1 Phi phi2 Inv. 71.4745 42.0163 292.388 0 192.611 27.4292 71.4178 0 {% endhighlight %}

It is important to understand that the reduction to 12 variants from 24 symmetrically equivalent parent orientations comes from the specific Burger orientation relationship. For a general orientation relationship, e.g., if we disturb the OR a little bit

{% highlight matlab %} beta2alpha = beta2alpha .* orientation.rand(csBeta,csBeta,'maxAngle',2*degree) {% endhighlight %} {% highlight plaintext %} beta2alpha = misorientation crystal symmetry : Ti (beta) (432) crystal symmetry : Ti (alpha) (622, X||a*, Y||b, Z||c*) Bunge Euler angles in degree phi1 Phi phi2 Inv. 184.879 90.9856 45.3841 0 {% endhighlight %}

we will always have exactly 24 variants. For the above example we observe how the 12 pairs of orientations which diverge slightly.

{% highlight matlab %} plotIPDF(variants(beta2alpha,oriParent),vector3d.Z) {% endhighlight %}
{% include inline_image.html file="ParentChildVariants_04.png" %}

Sometimes one faces the inverse question, i.e., determine the variantId from a parent and a child orientation or a pair of child orientations. This can be done using the command calcVariants which is discussed in detail in the section parent grain reconstruction.
