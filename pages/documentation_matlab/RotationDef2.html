This section describes the class rotation and gives an overview on how to work with rotations in MTEX.

Class Description

The class rotation is an inheritance of the class quaternion and allow to work with rotations as with matrixes in MTEX.

SUB: Euler Angle Conventions

There are several ways to specify a rotation in MTEX. A well-known possibility are the so called Euler angles. In texture analysis the following conventions are commonly used

  • Bunge (phi1,Phi,phi2) - ZXZ
  • Matthies (alpha,beta,gamma) - ZYZ
  • Roe (Psi,Theta,Phi)
  • Kocks (Psi,Theta,phi)
  • Canova (omega,Theta,phi)

Defining a Rotation by Bunge Euler Angles

The default Euler angle convention in MTEX are the Bunge Euler angles. Here a rotation is determined by three consecutive rotations, the first about the z-axis, the second about the y-axis, and the third again about the z-axis. Hence, one needs three angles to define an rotation by Euler angles. The following command defines a rotation by its three Bunge Euler angles

{% highlight matlab %} o = rotation.byEuler(30*degree,50*degree,10*degree) {% endhighlight %} {% highlight matlab %} o = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 30 50 10 0 {% endhighlight %}

Defining a Rotation by Other Euler Angle Conventions

In order to define a rotation by a Euler angle convention different to the default Euler angle convention you to specify the convention as an additional parameter, e.g.

{% highlight matlab %} o = rotation.byEuler(30*degree,50*degree,10*degree,'Roe') {% endhighlight %} {% highlight matlab %} o = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 120 50 280 0 {% endhighlight %}

Changing the Default Euler Angle Convention

The default euler angle convention can be changed by the command setpref, for a permanent change the mtex_settings should be edited. Compare

{% highlight matlab %} setMTEXpref('EulerAngleConvention','Roe') o {% endhighlight %} {% highlight matlab %} o = rotation size: 1 x 1 Roe Euler angles in degree Psi Theta Phi Inv. 30 50 10 0 {% endhighlight %} {% highlight matlab %} setMTEXpref('EulerAngleConvention','Bunge') o {% endhighlight %} {% highlight matlab %} o = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 120 50 280 0 {% endhighlight %}

SUB: Other Ways of Defining a Rotation

The axis angle parametrisation

A very simple possibility to specify a rotation is to specify the rotational axis and the rotational angle.

{% highlight matlab %} o = rotation.byAxisAngle(xvector,30*degree) {% endhighlight %} {% highlight matlab %} o = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 0 30 0 0 {% endhighlight %}

Four vectors defining a rotation

Given four vectors u1, v1, u2, v2 there is a unique rotations q such that q u1 = v1 and q u2 = v2.

{% highlight matlab %} o = rotation.map(xvector,yvector,zvector,zvector) {% endhighlight %} {% highlight matlab %} o = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 90 0 0 0 {% endhighlight %}

If only two vectors are specified the rotation with the smallest angle is returned that maps the first vector onto the second one.

{% highlight matlab %} o = rotation.map(xvector,yvector) {% endhighlight %} {% highlight matlab %} o = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 90 0 0 0 {% endhighlight %}

A fibre of rotations

The set of all rotations that rotate a certain vector u onto a certain vector v define a fibre in the rotation space. A discretisation of such a fibre is defined by

{% highlight matlab %} u = xvector; v = yvector; o = rotation(fibre(u,v)) {% endhighlight %} {% highlight matlab %} o = rotation size: 1000 x 1 {% endhighlight %}

Defining a rotation by a 3 times 3 matrix

{% highlight matlab %} o = rotation.byMatrix(eye(3)) {% endhighlight %} {% highlight matlab %} o = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 0 0 0 0 {% endhighlight %}

Defining a rotation by a quaternion

A last possibility is to define a rotation by a quaternion, i.e., by its components a,b,c,d.

{% highlight matlab %} o = rotation(quaternion(1,0,0,0)) {% endhighlight %} {% highlight matlab %} o = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 0 0 0 0 {% endhighlight %}

Actually, MTEX represents internally every rotation as a quaternion. Hence, one can write

{% highlight matlab %} q = quaternion(1,0,0,0) o = rotation(q) {% endhighlight %} {% highlight matlab %} q = Quaternion size: 1 x 1 a b c d 1 0 0 0 o = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 0 0 0 0 {% endhighlight %}

SUB: Calculating with Rotations

Rotating Vectors

Let

{% highlight matlab %} o = rotation.byEuler(90*degree,90*degree,0*degree) {% endhighlight %} {% highlight matlab %} o = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 90 90 0 0 {% endhighlight %}

a certain rotation. Then the rotation of the xvector is computed via

{% highlight matlab %} v = o * xvector {% endhighlight %} {% highlight matlab %} v = vector3d size: 1 x 1 x y z 0 1 0 {% endhighlight %}

The inverse rotation is computed via the backslash operator

{% highlight matlab %} o \ v {% endhighlight %} {% highlight matlab %} ans = vector3d size: 1 x 1 x y z 1 0 0 {% endhighlight %}

Concatenating Rotations

Let

{% highlight matlab %} rot1 = rotation.byEuler(90*degree,0,0); rot2 = rotation.byEuler(0,60*degree,0); {% endhighlight %}

be two rotations. Then the rotation defined by applying first rotation one and then rotation two is computed by

{% highlight matlab %} rot = rot2 * rot1 {% endhighlight %} {% highlight matlab %} rot = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 0 60 90 0 {% endhighlight %}

Computing the rotation angle and the rotational axis

Then rotational angle and the axis of rotation can be computed via then commands angle(rot) and axis(rot)

{% highlight matlab %} angle(rot)/degree axis(rot) {% endhighlight %} {% highlight matlab %} ans = 104.4775 ans = vector3d size: 1 x 1 x y z 0.447214 -0.447214 0.774597 {% endhighlight %}

If two rotations are specified the command angle(rot1,rot2) computes the rotational angle between both rotations

{% highlight matlab %} angle(rot1,rot2)/degree {% endhighlight %} {% highlight matlab %} ans = 104.4775 {% endhighlight %}

The inverse Rotation

The inverse rotation you get from the command inv(rot)

{% highlight matlab %} inv(rot) {% endhighlight %} {% highlight matlab %} ans = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 90 60 180 0 {% endhighlight %}

SUB: Conversion into Euler Angles and Rodrigues Parametrisation

There are methods to transform quaternion in almost any other parameterization of rotations as they are:

  • [[quaternion.Euler.html,Euler(rot)]] in Euler angle
  • [[quaternion.Rodrigues.html,Rodrigues(rot)]] % in Rodrigues parameter
{% highlight matlab %} [alpha,beta,gamma] = Euler(rot,'Matthies') {% endhighlight %} {% highlight matlab %} alpha = 4.7124 beta = 1.0472 gamma = 3.1416 {% endhighlight %}
