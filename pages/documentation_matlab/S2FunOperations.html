The idea of S2Fun is to calculate with spherical functions similarly as Matlab does with vectors and matrices. In order to illustrate this we consider the following two spherical functions

{% highlight matlab %} sF1 = S2Fun.smiley; sF2 = S2FunHarmonic.unimodal('halfwidth',10*degree) plot(sF1,'upper') nextAxis plot(sF2,'upper') {% endhighlight %} {% highlight plaintext %} sF2 = S2FunHarmonic bandwidth: 50 {% endhighlight %}
{% include inline_image.html file="S2FunOperations_01.png" %}

Basic arithmetic operations

Now the sum of these two spherical functions is again a spherical function

{% highlight matlab %} 1 + 15 * sF1 + sF2 plot(15 * sF1 + sF2,'upper') {% endhighlight %} {% highlight plaintext %} ans = S2FunHarmonic bandwidth: 128 {% endhighlight %}
{% include inline_image.html file="S2FunOperations_02.png" %}

Accordingly, one can use all basic operations like '-', '*', '^', '/', min, max, abs, sqrt to calculate with variables of type S2Fun.

{% highlight matlab %} % the maximum between two functions plot(max(15*sF1,sF2),'upper'); nextAxis % the minimum between two functions plot(min(15*sF1,sF2),'upper'); {% endhighlight %}
{% include inline_image.html file="S2FunOperations_03.png" %}

Local Extrema

The obove mentioned functions min and max have very different use cases

  • if two spherical functions are passed as arguments a spherical functions defined as the pointwise min/max between these two functions is computed
  • if a spherical function and single number are passed as arguments a spherical functions defined as the pointwise min/max between these the function and the value is computed
  • if only a single spherical function is provided the global maximum / minimum of the function is returned
  • if additionally the option 'numLocal' is provided the certain number of local minima / maxima is computed
{% highlight matlab %} plot(15 * sF1 + sF2,'upper') % compute and mark the global maximum [maxvalue, maxnodes] = max(15 * sF1 + sF2); annotate(maxnodes) % compute and mark the local minimum [minvalue, minnodes] = min(15 * sF1 + sF2,'numLocal',2); annotate(minnodes) {% endhighlight %}
{% include inline_image.html file="S2FunOperations_04.png" %}

Integration

The surface integral of a spherical function can be computed by either mean or sum. The difference between both commands is that sum normalizes the integral of the identical function on the sphere to {% include inline_image.html file="S2FunOperations_eq12586085190606786913.png" %} the command mean normalizes it to one. Compare

{% highlight matlab %} mean(sF1) sum(sF1) / ( 4 * pi ) {% endhighlight %} {% highlight plaintext %} ans = 0.0329 ans = 0.0329 {% endhighlight %}

A practical application of integration is the computation of the {% include inline_image.html file="S2FunOperations_eq11809360406450963432.png" %} -norm which is defined for a spherical function {% include inline_image.html file="S2FunOperations_eq18096895394918367257.png" %} as

{% include inline_image.html file="S2FunOperations_eq00716988343388260260.png" %}

accordingly we can compute it by

{% highlight matlab %} sqrt(sum(sF1.^2)) {% endhighlight %} {% highlight plaintext %} ans = 0.4138 {% endhighlight %}

or more efficiently by the command norm

{% highlight matlab %} norm(sF1) {% endhighlight %} {% highlight plaintext %} ans = 0.4138 {% endhighlight %}

Differentiation

The differential of a spherical function in a specific point is a gradient, i.e., a three-dimensional vector which can be computed by the command grad

{% highlight matlab %} grad(sF1,xvector) {% endhighlight %} {% highlight plaintext %} ans = vector3d size: 1 x 1 x y z 0 -0.000291054 -0.000490867 {% endhighlight %}

The gradients of a spherical function in all points form a spherical vector field and are returned by the function grad as a variable of type S2VectorFieldHarmonic

{% highlight matlab %} % compute the gradient as a vector field G = grad(sF1) % plot the gradient on top of the function plot(sF1,'upper') hold on plot(G) hold off {% endhighlight %} {% highlight plaintext %} G = S2VectorFieldHarmonic bandwidth: 129 {% endhighlight %}
{% include inline_image.html file="S2FunOperations_05.png" %}

We observe long arrows at the positions of big changes in intensity and almost invisible arrows in regions of constant intensity.

Rotating spherical functions

Rotating a spherical function works with the command rotate

{% highlight matlab %} % define a rotation rot = rotation.byAxisAngle(yvector,-30*degree) % plot the rotated spherical function plot(rotate(15 * sF1 + sF2,rot),'upper') {% endhighlight %} {% highlight plaintext %} rot = rotation size: 1 x 1 Bunge Euler angles in degree phi1 Phi phi2 Inv. 270 30 90 0 {% endhighlight %}
{% include inline_image.html file="S2FunOperations_06.png" %}

A special case of rotation is symmetrysing it with respect to some symmetry. The following example symmetrises our smiley with respect to a two fold axis in {% include inline_image.html file="S2FunOperations_eq17119888974628994297.png" %} direction

{% highlight matlab %} % define the symmetry cs = crystalSymmetry('112'); % compute the symmetrised function sFs = symmetrise(sF1, cs) % plot it plota2east plot(sFs,'upper','complete') {% endhighlight %} {% highlight plaintext %} sFs = S2FunHarmonic symmetry: 112, X||a, Y||b*, Z||c* bandwidth: 128 {% endhighlight %}
{% include inline_image.html file="S2FunOperations_07.png" %}

The resulting function is of type S2FunHarmonicSym and knows about its symmetry.
